Question 15 10 points

Draw an EER diagram for the following scenario:

A university database needs to track Students, Courses, and Instructors. Each student can enroll in multiple courses. Each course is taught by exactly one instructor, but an instructor can teach multiple courses. Students have a student ID (key), name, and email. Courses have a course code (key), title, and credits.

💡 Hint:

  • Start by creating the main entities (Student, Course, Instructor)
  • Don't forget to add the enrollment relationship between Student and Course
  • Include cardinality constraints (1:N, M:N relationships)

JavaScript Initialization

// Auto-initialization (recommended for test questions)
// Component auto-initializes on DOMContentLoaded

// Manual initialization
const container = document.getElementById('eer-diagram-question');
const component = new EERDQuestionComponent(container, {
    notation: 'chen',  // or 'ie'
    validationRules: { ... }
});

// Get answer data
const answerData = component.getAnswer();

// Validate
const issues = component.validate();

// Clear
component.clear();

// Destroy when done
component.destroy();

Supported Notations

  • Chen Notation: Rectangles for entities, ovals for attributes, diamonds for relationships
  • IE (Crow's Foot) Notation: Rounded rectangles, crow's foot symbols for cardinality

Element Types

  • Entities: Strong, Weak, IE-style
  • Attributes: Simple, Multi-valued, Derived, Key
  • Relationships: Regular, Identifying
  • Cardinality: 1, N, M, zero-one, one-many symbols
  • Subtypes: Triangle, Exclusive (d), Overlapping (o)
