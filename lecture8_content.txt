ERD Notations ( we learn both in this course )

Chen Notation
Good for learning core concepts. Has richer notation 
(e.g., we can model multi-valued attributes, composite 
attributes)
Not generally used in modern, applied use but if you 
know Chen notation concepts, you can generally adapt to 
use others easily.
Can be used for Conceptual and Logical Modeling. Not 
generally used for Physical Modeling

Information Engineering Notation
The most common notation used in an applied setting.
Can be used for Conceptual and Logical Modeling (less 
rich than Chen) . Idea for Physical Modeling.

Roles / Role Names in Modeling
A role or role name provides a clarifying label for the entity as it 
participates in a particular relationship.
We use this in everyday language.
An employee who is teaching a course is playing the role of “professor” or 
“instructor”
A person who is participating in a course to obtain credit towards a 
credential is playing the role of “student”.

Roles to the Rescue….
Consider a marketplace in which the same person can buy something or sell something.
Wrong
We should not model the user as two separate entities (a seller and a buyer) as this would create a 
cumbersome experience (”Why do I have to use a different login ID when I am buying versus 
selling?”)
Role-Based Model
Instead, we model a single user and we can describe their role in relation to a specific transaction.

Weak Entities
24
Weak entities describe a situation in which an entity relies on an 
attribute from a parent for its identity.
Real world data examples:
To uniquely identify a line item on an Amazon order, we need to know the 
order number (the order line item is a weak entity and the relation to order is 
identifying).
An employee database in which we record details of the dependents for 
each employee (children, spouse..etc.). The dependent cannot exist without 
a related employee. Each dependent record is identified using a composite 
key that includes the employee number.
Don’t Confuse Mandatory Participation with weakness
A car rental agreement is identified by the agreement number.  An 
agreement cannot exist without a relationship to customer.  This is 
mandatory participation, but the agreement is not a weak entity.
