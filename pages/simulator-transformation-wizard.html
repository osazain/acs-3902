❓ How to Use This Wizard

🎯 What is this Wizard?

The EER Transformation Wizard guides you through the 8-step process of converting Enhanced Entity-Relationship (EER) diagrams into physical relational database schemas. Each step focuses on a specific transformation rule.

📋 The 8 Transformation Steps

1

Map Regular Entities

Create tables for all regular (strong) entity types with their simple attributes.

2

Map Weak Entities

Create tables for weak entities, including the owner's primary key as a foreign key.

3

Handle 1:1 Relationships

Choose from 3 approaches: Foreign Key, Merge, or Junction Table.

4

Map 1:N Relationships

Add the parent's primary key as a foreign key in the child (N-side) table.

5

Map M:N Relationships

Create a junction table with foreign keys from both participating entities.

6

Resolve Multi-Valued Attributes

Create a separate table for each multi-valued attribute.

7

Map N-ary Relationships

Create a table with foreign keys from all participating entities (3+).

8

Transform Superclasses

Choose from 4 approaches: Roll-Up, Push-Down, Relate, or Flags.

🎮 How to Navigate

• Click any step in the progress bar to jump directly to that step
• Use Previous/Next buttons to move through the wizard
• Select different exercises to see different examples
• For Steps 3 and 8, choose different approaches to see alternative designs

💡

Pro Tip

Start with Exercise 1 (Employee & Department) if you are new to EER transformations. It covers the most common scenarios in a simple, easy-to-understand way.
