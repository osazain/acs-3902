🔄 CTE Structure (Lecture 6)

Non-Recursive CTE

WITH cte_name AS (
  SELECT ... FROM ... WHERE ...
)
SELECT * FROM cte_name;
  • Single query definition
  • Temporary result set
  • Exists only during execution

Recursive CTE

WITH RECURSIVE cte_name AS (
  -- Anchor member
  SELECT ... FROM ... WHERE ...

  UNION

  -- Recursive member
  SELECT ... FROM ...
  INNER JOIN cte_name ON ...
)
SELECT * FROM cte_name;
  • Anchor: starting point
  • UNION combines results
  • Recursive: references CTE
  • Stops when no new rows

Optimization Techniques (Lecture 7)

↓
Push Selections Down
Apply filters early
Move σ closer to leaves to reduce rows before joins
↓
Push Projections Down
Remove unused columns

CTE Writing Exercises

CTE Writing Tips

  • Non-recursive: Use for breaking complex queries into readable steps
  • Recursive: Use for hierarchical data (org charts, bill of materials)
  • Anchor member: Must be non-recursive, provides starting point
  • Recursive member: Must reference the CTE name in FROM clause
  • UNION vs UNION ALL: Use ALL to keep duplicates, eliminates extra sorting
