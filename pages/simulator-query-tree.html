🎯 Push-Down Optimization

Move SELECT (σ) operations down the tree closer to the source tables to reduce intermediate result sizes. This minimizes the data flowing through expensive join operations.

⚡ Restrictive First

Apply more restrictive conditions before less restrictive ones to minimize tuple counts early. The query optimizer uses selectivity estimates to reorder conditions automatically.

🔗 Join vs Cross Product

Use explicit JOIN operations instead of cross products followed by selections. Joins are much more efficient as they only combine matching rows.
