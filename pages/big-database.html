Product Catalog Analysis
Using DISTINCT, wildcards, and pattern matching
Easy
SELECT DISTINCT category_id, CASE WHEN price < 50 THEN 'Budget' WHEN price < 200 THEN 'Mid-Range' ELSE 'Premium' END AS price_tier, COUNT(*) AS product_count FROM products WHERE name LIKE '%Pro%' OR name LIKE '%Max%' GROUP BY category_id, price_tier HAVING COUNT(*) > 100 ORDER BY product_count DESC;
⏱️ Execution: 120ms
📊 Rows: 24
💰 Avg Price: $189.50

Lecture 3: Joins & Aggregates in Practice

Revenue by Category with Running Total
Complex JOIN with multiple tables and window functions
Hard
WITH category_revenue AS ( SELECT c.category_id, c.name AS category_name, SUM(oi.quantity * oi.price) AS total_revenue, COUNT(DISTINCT o.order_id) AS order_count FROM categories c JOIN products p ON c.category_id = p.category_id JOIN order_items oi ON p.product_id = oi.product_id JOIN orders o ON oi.order_id = o.order_id WHERE o.order_date >= '2024-01-01' GROUP BY c.category_id, c.name ) SELECT category_name, total_revenue, order_count, ROUND(total_revenue / order_count, 2) AS avg_order_value, SUM(total_revenue) OVER (ORDER BY total_revenue DESC) AS running_total FROM category_revenue ORDER BY total_revenue DESC;
⏱️ Execution: 890ms
📊 Rows: 42
💰 Total Revenue: $2.4B
Customers Without Orders (Churn Risk)
Using LEFT JOIN to find gaps in the data
Medium
SELECT c.customer_id, c.name, c.email, c.signup_date, EXTRACT(DAY FROM (CURRENT_DATE - c.signup_date)) AS days_since_signup FROM customers c LEFT JOIN orders o ON c.customer_id = o.customer_id WHERE o.order_id IS NULL AND c.signup_date < CURRENT_DATE - INTERVAL '30 days' ORDER BY c.signup_date ASC LIMIT 20;
⏱️ Execution: 2.3s
📊 Rows: 20
⚠️ Churn Risk: High

Challenge Exercises

1
Black Friday Analysis
Calculate total revenue, order count, and average order value for Black Friday 2024 (Nov 29). Include comparison to the previous Friday.
Use a CTE to calculate daily metrics, then use LAG() window function to compare with the previous week.
2
Customer Lifetime Value
Find the top 10 customers by total lifetime value, including their first purchase date, last purchase date, and total number of orders.
Use GROUP BY on customers, SUM on order totals, and MIN/MAX on order dates.
3
Product Affinity Analysis
Find products that are frequently bought together (appear in the same order). Show the top 10 product pairs.
Self-join the order_items table on order_id where product_id is different.
4
Inventory Restocking Alert
Identify products that need restocking: stock_level < 100 AND avg daily sales > 10 in the last 30 days.
Join products with a subquery calculating average daily sales from order_items.
