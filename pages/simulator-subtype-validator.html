"Is A" Test Validator

Determine if a subtype relationship passes the semantic "Is A" test

✓ Valid Subtypes

Must make semantic sense as a type of the parent

  • "House IS A Dwelling" ✓
  • "Student IS A Person" ✓
  • "Sedan IS A Vehicle" ✓

✗ Invalid Subtypes

Usually describe status/attributes, not types

  • "Active IS A Customer" ✗ (status)
  • "Premium IS A Member" ✗ (tier)
  • "Expensive IS A Product" ✗ (attribute)

Key Rule

Subtypes should have different attributes or relationships from each other. If they're just categories, they may be attributes instead.

Exclusive vs Overlapping

Identify whether subtypes are mutually exclusive or can overlap

📊 Visual Guide: Exclusive vs Overlapping Subtypes

🔒 EXCLUSIVE (Disjoint)
✕
Entity can be ONLY ONE subtype at a time
Chen notation: circle with 'd'
Crow's Foot: single circle with '✕'
Example:
A Dwelling is either a House, Condo, OR Apartment - never more than one
🔀 OVERLAPPING
✓
Entity can be MULTIPLE subtypes simultaneously
Chen notation: circle with 'o'
Crow's Foot: double circle with '○'
Example:
A Person can be BOTH a Student AND an Employee at the same time

🔒 Exclusive (Disjoint)

Entity can be ONLY ONE subtype

  • Chen: Circle with 'd'
  • IE: Circle with '✕'
  • Single circle symbol

🔀 Overlapping

Entity can be MULTIPLE subtypes

  • Chen: Circle with 'o'
  • IE: Circle with '○'
  • Double circle symbol

Examples

Exclusive: Dwelling = House OR Condo OR Apartment
Overlapping: Person = Student AND Employee
