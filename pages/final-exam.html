📅 Saturday, April 18⏱️ Study Session 10am-1pm🎯 Comprehensive

Exam Overview

Cumulative Exam

Covers all content covered during the term plus assigned reading from the text.

Notable Exclusions

Will NOT be tested:

  • Syntax for MongoDB commands
  • Trigger Syntax/Declaration (concept/comprehension CAN be tested)
📊

Exam Structure and Weighting

Section Type Weight Focus Areas
Section A Multiple Choice (~10-20 questions) ~25% Various Topics (textbook, notes, assignments)
Section B DDL/DML (short answer or problem-based) ~15-20% Write DDL to implement requirements, Create table/view, Constraints (FK, PK, Unique, Check), Domain declaration, Insert/Update, Indexes
Section C SQL Queries (sample data set, write the query) ~15-20% Subquery, CTE, Aggregate Queries, Outer Joins
Section D Data Modeling (EERD Chen OR Info Engineering) ~10% Create a model from requirements, Normalization Theory Test: 1NF, 2NF, 3NF, BCNF, 4NF (definitional/theory only), Transforming a Chen EERD to a Physical Design using Info Engineering
Section E Relational Algebra ~10-15% Relational Algebra & Query Tree - Translation to/from SQL, Optimization
💡

Study Tips

1 Practice Exam

Do it without looking things up at first to test yourself. Time yourself to simulate exam conditions.

2 SQL Work

Re-do prior assignments without using the internet or solutions. Focus on understanding the logic.

3 Reading

Short Reading Sessions (~30 minutes). Summarize by writing concise notes in your own words.

4 Modeling Skills

Practice questions within Exam Prep. Focus on Chen EERD and Info Engineering transformations.

5 Term Test 2 Recap

Watch the Term Test 2 Recap video (available this week) to review common mistakes and key concepts.

Key Topics to Review

SQL & Database Design
  • DDL: CREATE TABLE, ALTER TABLE, DROP TABLE
  • Constraints: Primary Key, Foreign Key, Unique, Check, Domain
  • DML: INSERT, UPDATE, DELETE
  • Queries: Subqueries, CTEs, Aggregate Functions, Outer Joins
  • Views and their applications
Data Modeling
  • EERD Notation (Chen and Information Engineering)
  • 8-Step Transformation Process
  • Superclass/Subclass handling
  • Normalization: 1NF, 2NF, 3NF, BCNF, 4NF
Relational Algebra
  • Basic Operations: Selection, Projection, Union, Difference, Cartesian Product
  • Additional Operations: Join, Division, Rename
  • Query Trees and SQL Translation
  • Query Optimization Principles
